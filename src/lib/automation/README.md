# Automation — محرك الأتمتة

> محرك قواعد (rule engine) لأتمتة العمليات التجارية — ملف واحد يُشغّل سير العمل.

## الملف

| الملف | الوظيفة |
|-------|---------|
| `engine.ts` | محرك تقييم القواعد وتنفيذ الإجراءات تلقائياً |

## كيف يعمل

```
حدث يقع (Event)
    │
    ▼
محرك الأتمتة ──► تقييم القواعد (Rules)
    │                    │
    │              هل الشرط مُحقّق؟
    │                    │
    ├─ نعم ──► تنفيذ الإجراء (Action)
    │                │
    │                ▼
    │           إرسال إشعار / إنشاء فاتورة / تحديث حالة
    │
    └─ لا ──► تجاوز
```

## القواعد والإجراءات المدعومة

| القاعدة (Trigger) | الإجراء (Action) |
|--------------------|------------------|
| فاتورة جديدة | إرسال إشعار WhatsApp |
| فاتورة متأخرة > 30 يوم | إرسال تذكير |
| رصيد منخفض | تنبيه مدير المالية |
| عميل جديد مسجّل | ترحيب تلقائي |
| دفعة مُستلمة | تحديث حالة الفاتورة |

## المدخل الرئيسي

```ts
import { AutomationEngine } from '@/lib/automation/engine';

const engine = new AutomationEngine();
await engine.evaluate(event);
```

## العلاقة مع النظام

- القواعد تُخزّن في قاعدة البيانات (جدول `AutomationRule`)
- الإجراءات تُنفّذ عبر BullMQ workers (غير محظورة)
- الإشعارات تمر عبر `src/lib/integrations/` (WhatsApp, Email)
