  1410→  1410→/// chosen candidate (e.g. matcher picked candidates[0] but employee confirmed
  1411→  1411→/// candidates[1]). These overrides feed back into the `historicalMatch` evidence
  1412→  1412→/// signal so the matcher LEARNS from human corrections over time.
  1413→  1413→///
  1414→  1414→/// Lifecycle:
  1415→  1415→///   1. matcher returns candidates[0] as winner (productId=A)
  1416→  1416→///   2. employee reviews, decides productId=B (candidates[1]) is correct
  1417→  1417→///   3. POST /api/products/match-override records:
  1418→  1418→///        inputText, inputNormalized, fromProductId=A, toProductId=B,
  1419→  1419→///        chosenAlias, auditId (link to original ProductMatchAudit decision)
  1420→  1420→///   4. Next time the SAME normalized input is matched, the matcher sees a past
  1421→  1421→///      override and boosts confidence for productId=B's aliases.
  1422→  1422→///
  1423→  1423→/// `inputNormalized` is the Arabic-normalized form (via normalizeArabic) so that
  1424→  1424→/// minor spelling variations still hit the same override record. The compound
  1425→  1425→/// index `(companySlug, inputNormalized)` is the primary lookup path.

  1430→  1430→  inputNormalized String   // normalizeArabic(inputText) — fast exact re-lookup
  1431→  1431→  fromProductId   Int?     // matcher's original winner (null if matcher said "new")
  1432→  1432→  toProductId     Int      // product the employee chose instead
  1433→  1433→  chosenAlias     String?  // alias text the employee confirmed
  1434→  1434→  auditId         Int?     // link to ProductMatchAudit row of the original decision
  1435→  1435→  reason          String?  // optional employee note ("brand mismatch", "wrong size")
  1436→  1436→  overriddenBy    String   // user email/id
  1437→  1437→  createdAt       DateTime @default(now())
  1438→  1438→
  1439→  1439→  tenant Company @relation(fields: [companySlug], references: [slug], onDelete: Restrict)
  1440→  1440→
  1441→  1441→  @@index([companySlug, inputNormalized])
  1442→  1442→  @@index([companySlug, toProductId])
  1443→  1443→  @@map("match_overrides")
  1444→  1444→}
