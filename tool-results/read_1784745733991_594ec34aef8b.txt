   647→### ⚠️ Cover Content Overflow Prevention (Mandatory)
   648→
   649→1. Cover section page margin is 0; total content height ≤ 15638 twips (1200 twips safety margin for cross-engine compatibility — MS Office renders large fonts taller than calculated).
   650→2. Color block Table `height` must use `rule: "exact"` (never `"atLeast"`).
   651→3. Each recipe code includes height budget annotations — verify during generation.
   652→4. **`verticalAlign` must always be `"top"`**. Never use `"center"` or `"bottom"` in exact-height rows — content will be clipped or overflow. Use `spacing.before` on the first paragraph for vertical positioning.
   653→5. **Title font size MUST be dynamically calculated via `calcTitleLayout()`** (see below). Never hardcode font sizes above 40pt for cover titles. Every recipe MUST call `calcTitleLayout()` before building title paragraphs.
   654→6. **Never use `margins.top`/`margins.bottom` for vertical positioning inside exact-height cells**: Cell margins reduce available height unpredictably across MS Office and WPS. Use `spacing.before` on the first paragraph instead. Only `margins.left`/`margins.right` are safe.
   655→7. **Dynamic spacing is mandatory**: Use `calcCoverSpacing()` to compute `spacing.before` values dynamically based on content element count and title line count. Never use fixed large spacing values (e.g., `before: 4500`) that assume a specific title length.
   656→8. **Cover must be a single-page section**: The cover section must produce exactly ONE page. If content overflows to a second page, it means the height budget is violated. Common overflow causes and fixes:
   657→   - Title font too large → use `calcTitleLayout()` to auto-reduce
   658→   - Too many meta lines → reduce font size or remove less important lines
   659→   - Fixed `spacing.before` values too large → use `calcCoverSpacing()` for dynamic values
   660→   - Subtitle + English label + meta lines combined exceed budget → calculate total and reduce spacing
   661→9. **Cover wrapper table MUST use explicit `allNoBorders`**: The outer 16838 wrapper table and ALL nested tables inside the cover MUST set borders to NONE explicitly. Never rely on docx-js default borders (`single/auto/sz=4`). Default borders add ~8 twips per edge, which causes MS Office to calculate a total height slightly exceeding 16838 → content overflows to a blank page 2. WPS is more lenient but MS Office is strict. **This is the #1 cause of "blank page 2 in MS Office but not in WPS".**
   662→
   663→```js
   664→// ✅ MANDATORY: Define and use allNoBorders for every cover table
   665→const NB = { style: BorderStyle.NONE, size: 0, color: "FFFFFF" };
   666→const noBorders = { top: NB, bottom: NB, left: NB, right: NB };
   667→const allNoBorders = { top: NB, bottom: NB, left: NB, right: NB,
   668→                       insideHorizontal: NB, insideVertical: NB };
   669→
   670→new Table({
   671→  borders: allNoBorders,  // ← MANDATORY on every cover table
   672→  // ...
   673→});
   674→```
   675→
   676→10. **Decorative lines MUST use paragraph borders, NEVER text characters**: Horizontal decorative lines (accent strips, dividers, frame edges) must be implemented with `paragraph border.top` or `border.bottom` — never with text characters like `───`, `━━━`, `═══`, or `——————`. Character-drawn lines render at inconsistent widths across MS Office and WPS (font metrics differ), causing lines to appear truncated or misaligned. Paragraph borders render pixel-perfect in both engines and their width is controlled precisely via `indent.left` / `indent.right`.
   677→
   678→```js
   679→// ✅ Correct — paragraph border (R2 style thick accent rule)
   680→new Paragraph({
   681→  indent: { left: 1000, right: 1000 },
   682→  border: { top: { style: BorderStyle.SINGLE, size: 18, color: P.accent, space: 20 } },
   683→  children: [],
   684→})
   685→
   686→// ❌ FORBIDDEN — text character line (renders inconsistently)
   687→new Paragraph({
   688→  children: [new TextRun({ text: "───────────────", color: P.accent })]
   689→})
   690→```
   691→9. **Post-generation overflow check (mandatory)**: After building cover children, estimate total height:
   692→   ```js
   693→   function estimateCoverHeight(elements) {
   694→     let total = 0;
   695→     for (const el of elements) {
   696→       if (el instanceof Table) {
   697→         // Sum row heights (exact rows) or estimate 400 twips per row
   698→         const rows = el.root?.[0]?.rows || [];
   699→         for (const row of rows) {
   700→           total += row.height?.value || 400;
   701→         }
   702→       } else if (el instanceof Paragraph) {
   703→         const fontSize = el.root?.[0]?.size || 24; // half-pts
   704→         // ★ Use pt * 11.5 (= half-pt * 10 * 1.15) for accurate single-spacing estimate
   705→         const lineHeight = Math.max(fontSize * 11.5, 276); // min 276 (default 12pt)
   706→         const spacingBefore = el.spacing?.before || 0;
   707→         const spacingAfter = el.spacing?.after || 0;
   708→         total += lineHeight + spacingBefore + spacingAfter;
   709→       }
   710→     }
   711→     return total;
   712→   }
   713→   // ★ Target: estimateCoverHeight(coverChildren) < 15638 (16838 - 1200 safety)
   714→   ```

   718→### ⚠️ Cover Title Layout — calcTitleLayout() (Mandatory for ALL Recipes)
   719→
   720→**Every cover recipe MUST use `calcTitleLayout()` to determine title font size and line breaks.** Hardcoding font sizes or passing the full title as a single TextRun is FORBIDDEN.
   721→
   722→**Every paragraph with font size > body text MUST set explicit line spacing** to prevent top clipping:
   723→```js
   724→// ★ MANDATORY: prevent inherited small line spacing from clipping large fonts
   725→spacing: { line: Math.ceil(titlePt * 23), lineRule: "atLeast", after: 100 }
   726→// Example: 36pt → line: 828; 44pt → line: 1012
   727→```
   728→Without this, the paragraph inherits body text line spacing (e.g., 560tw), which is shorter than the font height → top of characters gets clipped.
   729→
   730→```js
   731→/**
   732→ * Calculate safe font size and smart line breaks for cover titles.
   733→ * MUST be called by every recipe before building title paragraphs.
   734→ *
   735→ * @param {string}  title          Full title string
   736→ * @param {number}  maxWidthTwips  Available width for title text (twips, after subtracting margins/padding)
   737→ * @param {number}  preferredPt    Desired max font size in pt (default 40)
   738→ * @param {number}  minPt          Minimum allowed font size in pt (default 24)
   739→ * @returns {{ titlePt: number, titleLines: string[] }}
   740→ */
   741→function calcTitleLayout(title, maxWidthTwips, preferredPt = 40, minPt = 24) {
   742→  // Each CJK character width ≈ pt × 20 twips
   743→  const charWidth = (pt) => pt * 20;
   744→  const charsPerLine = (pt) => Math.floor(maxWidthTwips / charWidth(pt));
   745→
   746→  // Try from preferredPt downward until title fits in ≤ 3 lines
   747→  let titlePt = preferredPt;
   748→  let lines;
   749→  while (titlePt >= minPt) {
   750→    const cpl = charsPerLine(titlePt);
   751→    if (cpl < 2) { titlePt -= 2; continue; }
   752→    lines = splitTitleLines(title, cpl);
   753→    if (lines.length <= 3) break;
   754→    titlePt -= 2;
   755→  }
   756→
   757→  // If still > 3 lines at minPt, force 3 lines
   758→  if (!lines || lines.length > 3) {
   759→    const cpl = charsPerLine(minPt);
   760→    lines = splitTitleLines(title, cpl);
   761→    titlePt = minPt;
   762→  }
   763→
   764→  return { titlePt, titleLines: lines };
   765→}
   766→
   767→/**
   768→ * Smart Chinese title line-breaking — breaks at semantic boundaries, never mid-word.
   769→ *
   770→ * Rules:
   771→ * 1. Prefer breaking after particles, punctuation, connectors, underscores, spaces
   772→ * 2. Never split a compound word (e.g., "管理规范" must not become "管理规" + "范")
   773→ * 3. No single-character orphan on the last line — merge into previous line
   774→ * 4. If no good break point found within 60-130% of charsPerLine, break at charsPerLine
   775→ *
   776→ * @param {string} title        Full title string
   777→ * @param {number} charsPerLine Max characters per line at current font size
   778→ * @returns {string[]}          Array of line strings
   779→ */
   780→function splitTitleLines(title, charsPerLine) {
   781→  if (title.length <= charsPerLine) return [title];
   782→
   783→  // Characters that are safe break points (break AFTER these)
   784→  const breakAfter = new Set([
   785→    ...'，。、；：！？',              // CJK punctuation
   786→    ...'的与和及之在于为',            // CJK particles/prepositions
   787→    ...'-_—–·/',                     // connectors
   788→    ...' \t',                         // whitespace
   789→  ]);
   790→
   791→  const lines = [];
   792→  let remaining = title;
   793→
   794→  while (remaining.length > charsPerLine) {
   795→    let breakAt = -1;
   796→
   797→    // Search backward from charsPerLine to 60% for a break point
   798→    for (let i = charsPerLine; i >= Math.floor(charsPerLine * 0.6); i--) {
   799→      if (i < remaining.length && breakAfter.has(remaining[i - 1])) {
   800→        breakAt = i;
   801→        break;
   802→      }
   803→    }
   804→
   805→    // If not found, search forward up to 130%
   806→    if (breakAt === -1) {
   807→      const limit = Math.min(remaining.length, Math.ceil(charsPerLine * 1.3));
   808→      for (let i = charsPerLine + 1; i < limit; i++) {
   809→        if (breakAfter.has(remaining[i - 1])) {
   810→          breakAt = i;
   811→          break;
   812→        }
   813→      }
   814→    }
   815→
   816→    // Last resort: break at charsPerLine, but avoid splitting compound CJK words
   817→    if (breakAt === -1) {
   818→      breakAt = charsPerLine;
   819→      // If both chars at the break boundary are CJK (likely a compound word),
   820→      // step back 1 char to keep the word together
   821→      const prevChar = remaining[breakAt - 1];
   822→      const nextChar = remaining[breakAt];
   823→      if (prevChar && nextChar &&
   824→          !breakAfter.has(prevChar) && !breakAfter.has(nextChar) &&
   825→          /[\u4e00-\u9fff]/.test(prevChar) && /[\u4e00-\u9fff]/.test(nextChar)) {
   826→        breakAt = breakAt - 1;
   827→      }
   828→    }
   829→
   830→    lines.push(remaining.slice(0, breakAt).trim());
   831→    remaining = remaining.slice(breakAt).trim();
   832→  }
   833→  if (remaining) lines.push(remaining);
   834→
   835→  // Prevent single-character orphan on last line — merge into previous
   836→  if (lines.length > 1 && lines[lines.length - 1].length <= 2) {
   837→    const last = lines.pop();
   838→    lines[lines.length - 1] += last;
   839→  }
   840→
   841→  return lines;
   842→}
   843→
   844→/**
   845→ * Calculate dynamic spacing values for cover elements to fit within page height.
   846→ *
   847→ * @param {object} params
   848→ * @param {number} params.titleLineCount   Number of title lines
   849→ * @param {number} params.titlePt          Title font size in pt
   850→ * @param {boolean} params.hasSubtitle     Whether subtitle exists
   851→ * @param {boolean} params.hasEnglishLabel Whether English label exists
   852→ * @param {number} params.metaLineCount    Number of meta info lines
   853→ * @param {number} params.fixedHeight      Sum of fixed-height elements (color strips, accent bars, footer) in twips
   854→ * @param {number} params.pageHeight       Total page height in twips (default 16838)
   855→ * @returns {{ topSpacing, midSpacing, bottomSpacing }}  Spacing values in twips
   856→ */
   857→function calcCoverSpacing(params) {
   858→  const {
   859→    titleLineCount = 1, titlePt = 36, hasSubtitle = false,
   860→    hasEnglishLabel = false, metaLineCount = 0,
   861→    fixedHeight = 800, pageHeight = 16838,
   862→    marginTop = 0, marginBottom = 0,   // ★ NEW: pass actual section margins
   863→  } = params;
   864→
   865→  // ★ Safety margin: 1200 twips (cross-engine: MS Office renders large fonts
   866→  // taller than calculated; extra 400tw buffer prevents footer clipping)
   867→  const SAFETY = 1200;
   868→  // ★ Subtract page margins from available height (cover section may have margins)
   869→  const usableHeight = pageHeight - marginTop - marginBottom - SAFETY;
   870→
   871→  // ★ Accurate height estimation per element:
   872→  const titleHeight = titleLineCount * (titlePt * 23 + 200);
   873→  const subtitleHeight = hasSubtitle ? (12 * 23 + 600) : 0;
   874→  const englishLabelHeight = hasEnglishLabel ? (9 * 23 + 600) : 0;
   875→  const metaHeight = metaLineCount * (10 * 23 + 100);
   876→
   877→  // ★ Account for implicit paragraph heights:
   878→  const implicitParaHeight = 3 * 300;
   879→
   880→  const contentHeight = titleHeight + subtitleHeight + englishLabelHeight +
   881→                        metaHeight + fixedHeight + implicitParaHeight;
   882→
   883→  const remainingSpace = usableHeight - contentHeight;
   884→  const safeRemaining = Math.max(remainingSpace, 400);
   885→
   886→  // ★ Footer protection: bottomSpacing must be ≥ FOOTER_MIN to prevent
   887→  // footer + accent line from being clipped at the cell bottom edge
   888→  const FOOTER_MIN = 800;
   889→  const rawTop = Math.floor(safeRemaining * 0.45);
   890→  const rawBottom = Math.floor(safeRemaining * 0.45);
   891→  const bottomSpacing = Math.max(rawBottom, FOOTER_MIN);
   892→  const topSpacing = Math.max(rawTop - Math.max(0, FOOTER_MIN - rawBottom), 400);
   893→  const midSpacing = Math.max(safeRemaining - topSpacing - bottomSpacing, 0);
   894→
   895→  return { topSpacing, midSpacing, bottomSpacing };
   896→}
   897→```
   898→
   899→**Usage in every recipe:**
   900→```js
   901→// Step 1: Calculate title layout
   902→const availableWidth = 11906 - leftPadding - rightPadding; // subtract margins
   903→const { titlePt, titleLines } = calcTitleLayout(config.title, availableWidth);
   904→const titleSize = titlePt * 2; // convert to half-points for docx-js
   905→
   906→// Step 2: Calculate spacing (pass actual section margins!)
   907→const spacing = calcCoverSpacing({
   908→  titleLineCount: titleLines.length,
   909→  titlePt,
   910→  hasSubtitle: !!config.subtitle,
   911→  hasEnglishLabel: !!config.englishLabel,
   912→  metaLineCount: (config.metaLines || []).length,
   913→  fixedHeight: 800, // sum of accent strips, footer table, etc.
   914→  marginTop: 0,     // ★ pass the cover section's actual top margin (twips)
   915→  marginBottom: 0,  // ★ pass the cover section's actual bottom margin (twips)
   916→});
   917→
   918→// Step 3: Use in paragraphs
   919→children.push(new Paragraph({ spacing: { before: spacing.topSpacing } }));
   920→// ... title paragraphs using titleLines and titleSize ...
   921→children.push(new Paragraph({ spacing: { before: spacing.bottomSpacing } }));
   922→```

   966→
   967→**Visual:** Full-page dark background + left-aligned text + decoration via paragraph borders only

   981→// ⚠️ MANDATORY: Cover section must use margin: 0. See "Cover Section Non-Negotiables" above.
   982→// Section: { properties: { page: { size: { width: 11906, height: 16838 },

   984→
   985→function buildCoverR1(config) {
   986→  // config: { title, subtitle, englishLabel, metaLines, footerLeft, footerRight, palette }
   987→  // palette: { bg, titleColor, subtitleColor, metaColor, accent, footerColor }
   988→  const P = config.palette;
   989→  const padL = 1200, padR = 800;
   990→
   991→  // ⚠️ MANDATORY: Use calcTitleLayout() for dynamic font size + line breaking
   992→  const availableWidth = 11906 - padL - padR - 300; // -300 for border space
   993→  const { titlePt, titleLines } = calcTitleLayout(config.title, availableWidth, 40, 24);
   994→  const titleSize = titlePt * 2;
   995→
   996→  // ⚠️ MANDATORY: Use calcCoverSpacing() for dynamic spacing
   997→  const spacing = calcCoverSpacing({
   998→    titleLineCount: titleLines.length, titlePt,
   999→    hasSubtitle: !!config.subtitle, hasEnglishLabel: !!config.englishLabel,
  1000→    metaLineCount: (config.metaLines || []).length,
  1001→    fixedHeight: 400, // footer line only (no nested tables)
  1002→  });
  1003→
  1004→  const accentLeft = { style: BorderStyle.SINGLE, size: 8, color: P.accent, space: 12 };
  1005→  const children = [];
  1006→
  1007→  // 1. Top whitespace (dynamic)
  1008→  children.push(new Paragraph({ spacing: { before: spacing.topSpacing } }));
  1009→
  1010→  // 2. English label with accent bottom border
  1011→  if (config.englishLabel) {
  1012→    children.push(new Paragraph({
  1013→      indent: { left: padL, right: padR }, spacing: { after: 500 },
  1014→      border: { bottom: { style: BorderStyle.SINGLE, size: 6, color: P.accent, space: 8 } },
  1015→      children: [new TextRun({ text: config.englishLabel.split("").join("  "),
  1016→        size: 18, color: P.accent, font: { ascii: "Calibri", eastAsia: "SimHei" }, characterSpacing: 40 })],
  1017→    }));
  1018→  }
  1019→
  1020→  // 3. Main title (dynamic font size + smart line breaks)
  1021→  for (let i = 0; i < titleLines.length; i++) {
  1022→    children.push(new Paragraph({
  1023→      indent: { left: padL },
  1024→      spacing: { after: i < titleLines.length - 1 ? 100 : 300, line: Math.ceil(titlePt * 23), lineRule: "atLeast" },
  1025→      children: [new TextRun({ text: titleLines[i], size: titleSize, bold: true,
  1026→        color: P.titleColor, font: { eastAsia: "SimHei", ascii: "Arial" } })],
  1027→    }));
  1028→  }
  1029→
  1030→  // 4. Subtitle
  1031→  if (config.subtitle) {
  1032→    children.push(new Paragraph({
  1033→      indent: { left: padL }, spacing: { after: 800 },
  1034→      children: [new TextRun({ text: config.subtitle, size: 24, color: P.subtitleColor,
  1035→        font: { eastAsia: "Microsoft YaHei", ascii: "Arial" } })],
  1036→    }));
  1037→  }
  1038→
  1039→  // 5. Meta info lines with left accent border
  1040→  for (const line of (config.metaLines || [])) {
  1041→    children.push(new Paragraph({
  1042→      indent: { left: padL + 200 }, spacing: { after: 80 },
  1043→      border: { left: accentLeft },
